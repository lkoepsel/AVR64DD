# Claude analysis of sysclock timing loop (orginally for ATtiny13A)

* Count = 30,000 => ~100 ms window
* OCR0A = 0x95 or 149
* Prescaler = /8

Result = 0x65 or 101

The ISR uses `r2` as a dedicated scratch register (reserved globally вҖ” see the
register notes at the top of `main.S`), so it needs no `push`/`pop`. That keeps the per-interrupt overhead to 15 cycles.

  On the rare carry path (once every 256 ticks, when `r8` wraps from 0xFF to
  0x00), `brne` falls through (1 cycle) and `inc r9` executes (1 cycle) вҖ” the
  same 15-cycle total.

  Because `r2` and the counter `r9:r8` are reserved registers, the handler skips the usual `push`/`pop` pair вҖ” an r16-based handler that saved/restored r16 on the stack would add 4 cycles (push + pop) for a total of 19 cycles; this one costs 15.

  Putting it together

  Wall-clock time of the loop = loop instructions + every ISR that fires during it:

  T_wall = 119,999 + 15 В· n      where  n = T_wall / 1200

  Solving the fixed point:

  T_wall = 119,999 / (1 вҲ’ 15/1200) = 119,999 Г— 1200/1185 вүҲ 121,518 cycles
  n      = 121,518 / 1200 вүҲ 101.3 ticks
